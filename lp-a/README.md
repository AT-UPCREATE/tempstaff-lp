# プロジェクト概要

このプロジェクトは、FigmaデザインをもとにWebサイトを実装するプロジェクトです。

目的は、見た目が綺麗で、あとから修正しやすいWebサイトを作ることです。

## サイトの目的

- サービスや事業の魅力をわかりやすく伝える
- ユーザーに信頼感を与える
- 問い合わせ・資料請求・予約などのアクションにつなげる
- PC・スマホのどちらでも見やすいWebサイトにする

## 実装方針

- Figmaのデザインをできるだけ再現する
- ただし、保守しにくい実装にならないように整理する
- 余白、文字サイズ、色、レスポンシブを丁寧に調整する
- あとから人間が修正しやすいコードにする

## ファイル構成

基本構成は以下を想定しています。

```txt
index.html
style.css
components/
sections/
assets/
README.md
AGENTS.md
```


ただし、すでに別の構成で実装されている場合は、既存構成を優先してください。

## 各ファイル・フォルダの役割

- `index.html`：トップページのHTML
- `style.css`：全体のスタイル
- `components/`：ボタン、カード、ヘッダーなどの共通パーツ
- `sections/`：Hero、About、Service、CTAなどのセクション
- `assets/`：画像、アイコン、フォントなどの素材
- `README.md`：プロジェクトの説明
- `AGENTS.md`：Codexへの作業ルール

## 確認時のポイント

- Figmaのデザインに近い見た目になっているか
- 余白や文字サイズに統一感があるか
- PCだけでなくスマホでも自然に見えるか
- 見出し構造が正しいか
- 画像が不自然に切れていないか
- ボタンやリンクが押しやすいか
- コードがあとから修正しやすいか

## 注意事項

- 既存コードを大きく壊さない
- 全体を一気に作り直さない
- セクションごとに小さく実装・調整する
- 変更した内容は作業後にまとめる
